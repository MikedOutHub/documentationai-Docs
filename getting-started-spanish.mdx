## Descripción general

Usa la biblioteca de pódcast de cstore para convertir los episodios del pódcast de C-Store Center en material de capacitación escrito que puedas ojear, compartir y reutilizar con tu equipo.

La biblioteca vive como archivos Markdown simples en un repositorio público de GitHub. Este sitio de documentación puede ir mostrando esos mismos archivos como páginas con el tiempo, pero el repositorio de GitHub siempre está completo y actualizado.

<Callout kind="info" collapsed="false">
  La fuente de la verdad para todas las transcripciones es el repositorio público de GitHub llamado `cstore-podcast-library`. Este sitio de documentación puede mostrar esos mismos archivos Markdown para que los leas sin salir del navegador, pero el contenido nuevo siempre aparece primero en GitHub.
</Callout>

## Qué hay hoy en la biblioteca

La biblioteca se centra actualmente en transcripciones y guías extensas basadas en el contenido del pódcast de C-Store Center.

Ahora mismo encontrarás al menos un recurso sustancial:

- **`navigating_convenience_store_industry.md`**: una transcripción y guía extensa, con secciones, que cubre temas como exhibición visual, inventario, capacitación del personal, precios, servicio al cliente y participación comunitaria.

Puedes leer esa guía directamente en el repositorio de GitHub en [navigating\_convenience\_store\_industry.md](https://github.com/MikedOutHub/cstore-podcast-library/blob/main/navigating_convenience_store_industry.md). Si existe una página equivalente en este sitio de documentación, muestra el mismo Markdown subyacente.

## Empieza a usar la biblioteca en tres pasos

Sigue estos pasos para acceder al contenido y sacarle provecho rápidamente.

<Steps>
  <Step title="Abre las transcripciones" icon="book-open" title-type="p">
    Si quieres leer en tu navegador directamente desde GitHub, abre el archivo `navigating_convenience_store_industry.md` en:

    [navigating\_convenience\_store\_industry.md en GitHub](https://github.com/MikedOutHub/cstore-podcast-library/blob/main/navigating_convenience_store_industry.md)

    Usa la búsqueda en la vista de archivos de GitHub (`<kbd>Ctrl</kbd> + <kbd>F</kbd>` o `<kbd>Cmd</kbd> + <kbd>F</kbd>`) para saltar a temas como `inventory` o `training`.

    Si prefieres trabajar con archivos de forma local, clona el repositorio de GitHub en tu computadora.

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      git clone https://github.com/MikedOutHub/cstore-podcast-library.git
      cd cstore-podcast-library
      ```

      ```bash
      gh repo clone MikedOutHub/cstore-podcast-library
      cd cstore-podcast-library
      ```
    </CodeGroup>

    Cuando termine la clonación, deberías ver los archivos Markdown en la nueva carpeta `cstore-podcast-library`.
  </Step>

  <Step title="Comprende la estructura y organización de archivos" icon="folder" title-type="p">
    Cada transcripción o guía es un archivo `.md` independiente en la carpeta raíz del repositorio, con encabezados que dividen el contenido en secciones prácticas.

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      ls
      README.md
      navigating_convenience_store_industry.md
      ```
    </CodeGroup>

    Abre `navigating_convenience_store_industry.md` en cualquier visor o editor de Markdown, o en la propia interfaz de GitHub. Encabezados como `Visual merchandising`, `Inventory and purchasing` y `Team training` funcionan como capítulos para que puedas ojear y saltar a lo que necesitas.
  </Step>

  <Step title="Convierte ideas en acciones con tu equipo" icon="users" title-type="p">
    Empieza con una sección que se relacione con una prioridad actual en tu tienda, como mejorar la exhibición o ajustar el inventario.

    Mientras leas esa sección:

    - Resalta o anota **acciones claras** (por ejemplo, "recorrer cada pasillo semanalmente para revisar las frentes de producto").

    - Reescribe esas acciones como una **lista de verificación de un POE** breve en tu herramienta interna (Google Docs, Notion o tu manual de operaciones).

    - Usa una o dos preguntas de la sección como **disparadores de conversación** para tu próxima reunión de equipo.

    Tu objetivo no es terminar toda la transcripción de una vez. Extrae una mejora enfocada y conviértela en un hábito repetible para tu tienda.
  </Step>
</Steps>

## Cómo leer y ojear transcripciones de forma efectiva

Las transcripciones y guías extensas pueden ser densas, así que concéntrate en la navegación y en extraer ideas, no en leer cada palabra.

Usa estas tácticas:

- **Ojear primero los encabezados.** Revisa los encabezados principales y secundarios para ver qué cubre la guía y luego elige una o dos secciones que coincidan con tus problemas actuales.

- **Saltar usando la búsqueda.** Usa la búsqueda (`inventory`, `training`, `pricing`) en tu navegador o editor para llegar directamente a los párrafos relevantes.

- **Marcar ejemplos.** Cuando veas ejemplos concretos (como cómo una tienda gestiona eventos comunitarios), anota qué podrías adaptar en lugar de copiar cada detalle.

- **Capturar acciones, no citas.** Escribe lo que harás de forma diferente en tu tienda, no solo frases memorables del pódcast.

Si conviertes cada sección útil en una pequeña lista de verificación o POE, la biblioteca se convierte en tu manual interno, no solo en algo que lees una vez.

## Formas de reutilizar el contenido internamente

Usa las transcripciones como materia prima para la capacitación y la alineación de todo tu equipo.

Algunos formatos que funcionan bien:

- **Módulos de capacitación para el equipo.** Asigna una sola sección (por ejemplo, "Conceptos básicos de servicio al cliente") como lectura previa y luego repasa las acciones clave en una reunión breve.

- **Disparadores para agendas de reuniones.** Copia dos o tres preguntas o ideas de una sección para estructurar un recorrido por la tienda, una reunión de gerentes o un huddle.

- **Onboarding de nuevas contrataciones.** Extrae las partes más prácticas (como cómo acomodar las estanterías o cómo saludar a los clientes) en una lista de verificación sencilla para nuevos miembros del equipo.

- **Proyectos de mejora de la tienda.** Elige un tema por mes (exhibición, inventario, capacitación) y usa las secciones relevantes de la transcripción como referencia durante ese mes.

No necesitas herramientas especiales para hacer esto. Copia, pega y adapta el texto a los sistemas que tu equipo ya utiliza.

## Contribuir una nueva transcripción o guía (opcional)

Si quieres agregar la transcripción de otro pódcast o una guía de capacitación relacionada, sigue un patrón sencillo para mantener la coherencia de la biblioteca.

<Steps>
  <Step title="Crea un nuevo archivo Markdown" icon="plus" title-type="p">
    Desde tu clon local del repositorio, agrega un nuevo archivo `.md` en la carpeta raíz. Usa un nombre de archivo claro y descriptivo con guiones bajos, por ejemplo:

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      cd cstore-podcast-library
      touch building_high_performance_store_teams.md
      ```
    </CodeGroup>

    Añade una breve descripción al inicio y usa encabezados para dividir el contenido en secciones prácticas (por ejemplo, `Overview`, `Hiring`, `Training`, `Coaching`).
  </Step>

  <Step title="Mantén una estructura coherente" icon="settings" title-type="p">
    Alinea el estilo general con `navigating_convenience_store_industry.md`:

    - Empieza con una **descripción general** que explique para quién es el contenido y qué cubre.

    - Usa **encabezados claros y orientados a la acción** en lugar de títulos ingeniosos.

    - Prefiere **párrafos cortos y listas con viñetas** para consejos y pasos.

    La coherencia facilita que otros dueños de tiendas y miembros del equipo puedan ojear y reutilizar tu contenido.
  </Step>

  <Step title="Abre un pull request" icon="github" title-type="p">
    Cuando tu nuevo archivo esté listo:

    <CodeGroup show-lines="true" tabs={[]}>
      ```bash
      git checkout -b add-building-high-performance-store-teams
      git add building_high_performance_store_teams.md
      git commit -m "Add building high performance store teams guide"
      git push origin add-building-high-performance-store-teams
      ```
    </CodeGroup>

    Luego abre un pull request en GitHub proponiendo la nueva transcripción o guía. Incluye una nota breve sobre de qué episodio o tema del pódcast proviene y qué problemas ayuda a resolver a los dueños de tiendas.
  </Step>
</Steps>
